# المساهمة | Contributing

شكراً لاهتمامك بالمساهمة في **إضافة إدارة الحساب** — أحد مشاريع الكِندِي مفتوح.
Thanks for your interest in contributing to the **Account Plug-in** — an AlKindy OSS project.

---

## 🇸🇦 العربية

### قبل البدء
- اقرأ [README.md](README.md) و[ميثاق السلوك](CODE_OF_CONDUCT.md)
- ابحث في [Issues](../../issues) قبل فتح طلب جديد

### سير العمل
1. Fork للمشروع
2. أنشئ branch جديد: `git checkout -b feature/<اسم-وصفي>`
3. ابنِ الإضافة محلياً: `npm install && npm run build`
4. التزم بمعايير الكود في المشروع
5. ارفع PR مع شرح واضح للتغيير

### Developer Certificate of Origin (DCO)
نستخدم DCO بدلاً من CLA. كل commit يجب أن يحتوي:
```
Signed-off-by: اسمك الكامل <your@email.com>
```
استخدم: `git commit -s`

بهذا التوقيع تؤكد أنك تملك حقوق المساهمة وتوافق على ترخيصها بـ Apache 2.0.

### معايير الـCommit
نتبع [Conventional Commits](https://www.conventionalcommits.org):
```
feat: إضافة ميزة جديدة
fix: إصلاح خطأ
docs: تحديث توثيق
test: إضافة اختبارات
refactor: إعادة هيكلة دون تغيير سلوك
```

## 🇬🇧 English

### Before You Start
- Read [README.md](README.md) and [Code of Conduct](CODE_OF_CONDUCT.md)
- Search [Issues](../../issues) before opening a new one

### Workflow
1. Fork the project
2. Create a branch: `git checkout -b feature/<descriptive-name>`
3. Build locally: `npm install && npm run build`
4. Follow project code standards
5. Submit a PR with clear description

### Developer Certificate of Origin (DCO)
We use DCO instead of CLA. Every commit must include:
```
Signed-off-by: Your Full Name <your@email.com>
```
Use: `git commit -s`

By signing off, you certify you authored or have rights to the contribution and agree to license it under Apache 2.0.

### Commit Standards
Follow [Conventional Commits](https://www.conventionalcommits.org):
```
feat: add new feature
fix: bug fix
docs: documentation update
test: add tests
refactor: code restructure without behavior change
```
