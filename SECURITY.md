# سياسة الأمان | Security Policy

## 🇸🇦 العربية

### الإصدارات المدعومة
| الإصدار | الدعم الأمني |
|---|---|
| 0.1.x | ✅ |
| < 0.1 | ❌ |

### الإبلاغ عن ثغرة
**لا تفتح Issue عاماً للثغرات الأمنية.**

راسلنا على: **`security@alkindy.tech`**

اشمل في البلاغ:
- وصف الثغرة وتأثيرها
- خطوات إعادة الإنتاج (PoC إن أمكن)
- الإصدار المتأثر
- مقترح إصلاح إن وجد

### وقت الاستجابة
- **إقرار الاستلام:** خلال 48 ساعة
- **تقييم أولي:** خلال 7 أيام
- **إصلاح للحرج:** ≤ 30 يوم
- **إفصاح منسّق:** نتفق معك على التوقيت

نقدّر الباحثين في الأمن ونذكرهم في `SECURITY-CREDITS.md` (إن رغبوا).

## 🇬🇧 English

### Supported Versions
| Version | Security Support |
|---|---|
| 0.1.x | ✅ |
| < 0.1 | ❌ |

### Reporting a Vulnerability
**Do not open public issues for security vulnerabilities.**

Email us at: **`security@alkindy.tech`**

Include in your report:
- Vulnerability description and impact
- Reproduction steps (PoC if possible)
- Affected version
- Suggested fix if any

### Response Time
- **Acknowledgement:** within 48 hours
- **Initial assessment:** within 7 days
- **Critical fix:** ≤ 30 days
- **Coordinated disclosure:** timing agreed with reporter

We credit security researchers in `SECURITY-CREDITS.md` (if they consent).
